 # LEARNINGS

## Day 11 – Project Planning & Setup

### What was completed

- Selected the Bank Customer Churn dataset.
- Created the project repository and folder structure.
- Initialized Git for version control.
- Created the project environment using uv.
- Installed project dependencies.
- Initialized DVC.
- Added the raw dataset to DVC tracking.
- Prepared the project proposal.
- Designed the system architecture.
- Created the project backlog.

---

### Key Decisions

- Selected a customer churn prediction problem because it is highly relevant in the banking industry.
- Chose F1 Score as the primary evaluation metric due to the class imbalance in churn prediction.
- Used DVC for dataset versioning.
- Planned MLflow for experiment tracking.
- Chose FastAPI for model serving.
- Planned Docker and Google Cloud Run for deployment.

---

### Challenges

- Selecting a suitable large-scale dataset.
- Choosing a dataset that balances engineering complexity with the project deadline.

---

### Lessons Learned

- Proper project planning reduces implementation issues.
- Data versioning is essential for reproducible machine learning.
- Repository structure plays an important role in project maintainability.

---

### Future Improvements

- Implement automated data validation.
- Add model drift monitoring.
